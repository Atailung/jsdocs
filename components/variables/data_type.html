Data Types

JavaScript variables can hold different types of data. Understanding these types is crucial for effective programming.

Primitive Types

Primitive values are immutable (cannot be changed) and are passed by value.

  • String

    Textual data enclosed in quotes. 

    let name = 'John';
  • Number

    Numeric values, including integers and floating-point. 

    let age = 25;
                    let price = 19.99;
  • Boolean

    Logical values: true or false. 

    let isActive = true;
  • Undefined

    A variable that has been declared but not assigned a value. 

    let unassigned; // value is undefined
  • Null

    Represents the intentional absence of any value. 

    let empty = null;
  • Symbol

    Unique and immutable primitive value. 

    let id = Symbol('id');
  • BigInt

    For integers larger than the Number type can handle. 

    let bigNumber = 9007199254740991n;

Reference Types

Reference values are mutable (can be changed) and are passed by reference.

  • Object

    Collection of key-value pairs. 

    let person = {
                      name: 'John',
                      age: 25
                    };
  • Array

    Ordered collection of values. 

    let colors = ['red', 'green', 'blue'];
  • Function

    Reusable block of code.

    let greet = function(name) {
                      return `Hello, ${name}!`;
                    };
  • Date

    Represents a specific point in time. 

    let today = new Date();
  • RegExp

    Regular expression for pattern matching. 

    let pattern = /\d+/g;
  • Map & Set

    Collections with unique keys/values. 

    let map = new Map();
                    let set = new Set();

Type Coercion & Checking

JavaScript is a dynamically typed language, which means variables can change types. Understanding type coercion and how to check types is important.

javascript
// Type coercion examples
                console.log('5' + 2);        // "52" (string concatenation)
                console.log('5' - 2);        // 3 (numeric subtraction)
                console.log('5' * 2);        // 10 (numeric multiplication)
                console.log(true + 1);       // 2 (true is coerced to 1)
                console.log(false + 1);      // 1 (false is coerced to 0)

                // Checking types
                let num = 42;
                let str = 'Hello';
                let obj = { name: 'John' };

                console.log(typeof num);  // "number"
                console.log(typeof str);  // "string"
                console.log(typeof obj);  // "object"

                // Array checking (typeof returns "object" for arrays)
                let arr = [1, 2, 3];
                console.log(typeof arr);                // "object"
                console.log(Array.isArray(arr));              // true

                // Null checking
                console.log(typeof null);                // "object" (this is a known JavaScript quirk)
                console.log(myVar === null);        // Direct comparison is better
