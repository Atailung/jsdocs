let vs const vs var

Understanding the differences between let, const, and var is crucial for writing clean, bug-free JavaScript code.

Feature let const var
Scope Block Block Function
Reassignment Yes No Yes
Redeclaration No No Yes
Hoisting Yes, but in TDZ Yes, but in TDZ Yes, initialized as undefined
Temporal Dead Zone Yes Yes No
Global Property No No Yes

Key Differences Explained

Scope

let and const are block-scoped, meaning they are only accessible within the block (denoted by {}) where they are defined. var is function-scoped, meaning it's accessible throughout the entire function it's defined in.

Reassignment

Variables declared with let and var can be reassigned new values. Variables declared with const cannot be reassigned after initialization.

Hoisting & Temporal Dead Zone

All declarations are hoisted, but let and const are in a "Temporal Dead Zone" (TDZ) until their declaration is reached. var is hoisted and initialized with undefined. (The hoisted allow variables desclared with var to be accessed before their decalration, but with the caveat of an initial value of undefined)

javascript
      
        // Scope differences
        {
          let blockScoped = 'only available in this block';
          const alsoBlockScoped = 'same as let';
          var notBlockScoped = 'available outside this block';
        }

        // console.log(blockScoped); // ReferenceError
        // console.log(alsoBlockScoped); // ReferenceError
        console.log(notBlockScoped); // "available outside this block"

        // Hoisting and TDZ
