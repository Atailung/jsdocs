Naming Conventions

Using consistent naming conventions makes your code more readable and maintainable. Here are the common conventions used in JavaScript.

Basic Rules

  • Must be valid identifiers

    Can contain letters, digits, underscores, and dollar signs

  • Must start with a letter, underscore, or dollar sign

    Cannot start with a number

  • Cannot be reserved keywords

    Words like if, for, function, etc.

  • Case-sensitive

    myVar and myvar are different variables

Common Conventions

  • camelCase for variables and functions

    firstName, calculateTotal()

  • PascalCase for classes and constructors

    Person, UserProfile

  • UPPER_SNAKE_CASE for constants

    MAX_USERS, API_KEY

  • Use descriptive names

    Prefer userAge over a

javascript
// Good variable naming examples

              // camelCase for variables
              let firstName = 'John';
              let userAge = 25;
              let isActive = true;

              // camelCase for functions
              function calculateTotal(price, quantity) {
                return price * quantity;
              }

              // PascalCase for classes
              class User {
                constructor(name, email) {
                  this.name = name;
                  this.email = email;
                }
              }

              // UPPER_SNAKE_CASE for constants
              const MAX_USERS = 100;
              const API_BASE_URL = 'https://api.example.com';

              // Bad naming examples (avoid these)
              let x = 'John';           // Not descriptive
              let user_name = 'John';    // Inconsistent with camelCase
              let UserAge = 25;          // PascalCase for a variable

Semantic Naming

Good variable names should clearly communicate their purpose and content. They should be:

  • Descriptive: Clearly indicate what the variable represents
  • Concise: Not unnecessarily long
  • Pronounceable: Easy to discuss in conversation
  • Consistent: Follow the same patterns throughout your codebase
