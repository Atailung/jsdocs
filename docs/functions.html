Function Basics

Functions are one of the fundamental building blocks in JavaScript. A function is a reusable block of code designed to perform a particular task.

Why Functions Matter

Functions allow us to:

  • Write reusable code that can be called multiple times
  • Break down complex problems into smaller, manageable pieces
  • Organize code in a modular, maintainable way
  • Create abstractions that hide implementation details
Input

Arguments

Function

Processes the inputs

Output

Return value

Think of a function as a machine: you give it some input (arguments), it processes that input according to a set of instructions, and then it produces an output (return value).

javascript
// A simple function that adds two numbers
function add(a, b) {
  return a + b;
}

// Calling the function
const sum = add(5, 3);
console.log(sum); // 8

Function Declaration

A function declaration (also called a function statement) defines a named function using the function keyword.

javascript
// Function declaration syntax
function functionName(parameter1, parameter2, ...) {
  // Function body - code to be executed
  return result; // Optional return statement
}

// Example of a function declaration
function greet(name) {
  return `Hello, ${name}!`;
}

// Calling the function
console.log(greet('John')); // "Hello, John!"

Key Characteristics

  • Hoisting

    Function declarations are hoisted to the top of their scope, meaning you can call the function before it's declared in your code.

  • Named Function

    The function has a name that can be used to call it from anywhere in its scope.

  • Function Scope

    Creates its own scope for variables declared inside it.

javascript
// Hoisting demonstration
console.log(hoistedFunction()); // "I am hoisted!"

// This works because function declarations are hoisted
function hoistedFunction() {
  return "I am hoisted!";
}

Function Expression

A function expression defines a function as part of a larger expression, typically a variable assignment.

javascript
// Function expression syntax
const functionName = function(parameter1, parameter2, ...) {
  // Function body
  return result;
};

// Example of a function expression
const greet = function(name) {
  return `Hello, ${name}!`;
};

// Calling the function
console.log(greet('Jane')); // "Hello, Jane!"

Key Characteristics

  • No Hoisting

    Function expressions are not hoisted, so you can only call them after they've been defined in your code.

  • Anonymous or Named

    Can be anonymous (as shown above) or named (e.g., const greet = function sayHello(name) {...}).

  • First-Class Objects

    Can be passed as arguments, returned from other functions, and assigned to variables.

javascript
// No hoisting demonstration
// console.log(notHoisted()); // Error: notHoisted is not a function

const notHoisted = function() {
  return "I am not hoisted!";
};

console.log(notHoisted()); // "I am not hoisted!"

// Named function expression
const factorial = function calculateFactorial(n) {
  if (n <= 1) return 1;
  return n * calculateFactorial(n - 1);
};

console.log(factorial(5)); // 120

Arrow Functions

Arrow functions, introduced in ES6 (ECMAScript 2015), provide a more concise syntax for writing function expressions.

javascript
// Arrow function syntax
const functionName = (parameter1, parameter2, ...) => {
  // Function body
  return result;
};

// Example of an arrow function
const greet = (name) => {
  return `Hello, ${name}!`;
};

// Calling the function
console.log(greet('Alex')); // "Hello, Alex!"

Concise Syntax

Arrow functions can be even more concise in certain situations:

Single Parameter

// Parentheses are optional with a single parameter
const square = x => x * x;

Implicit Return

// Curly braces and return are optional for single expressions
const add = (a, b) => a + b;

No Parameters

// Empty parentheses for no parameters
const sayHello = () => 'Hello, world!';

Returning Object Literal

// Wrap object literal in parentheses for implicit return
const createPerson = (name, age) => ({ name, age });

Key Differences from Regular Functions

  • No this Binding

    Arrow functions don't have their own this value. They inherit this from the enclosing scope.

  • No arguments Object

    Arrow functions don't have their own arguments object. Use rest parameters instead.

  • Cannot Be Used as Constructors

    Arrow functions cannot be used with the new keyword.

  • No super or new.target

    Arrow functions don't have access to super or new.target keywords.

javascript
// this binding difference
function RegularFunction() {
  this.value = 42;
  
  // Regular function: 'this' refers to the function's own context
  setTimeout(function() {
    console.log('Regular function:', this.value); // undefined (or window.value in browsers)
  }, 1000);
  
  // Arrow function: 'this' is inherited from the enclosing scope
  setTimeout(() => {
    console.log('Arrow function:', this.value); // 42
  }, 1000);
}

const obj = new RegularFunction();

Parameters & Arguments

Parameters are the names listed in the function definition, while arguments are the actual values passed to the function when it is called.

javascript
// Parameters: a, b
function multiply(a, b) {
  return a * b;
}

// Arguments: 5, 3
const result = multiply(5, 3);
console.log(result); // 15

Default Parameters

ES6 introduced default parameters, allowing you to specify default values for parameters that are not provided. 

function greet(name = 'Guest', greeting = 'Hello') {
  return `${greeting}, ${name}!`;
}

console.log(greet()); // "Hello, Guest!"
console.log(greet('John')); // "Hello, John!"
console.log(greet('John', 'Hi')); // "Hi, John!"

Rest Parameters

Rest parameters allow a function to accept an indefinite number of arguments as an array. 

function sum(...numbers) {
  return numbers.reduce((total, num) => total + num, 0);
}

console.log(sum(1, 2, 3, 4, 5)); // 15
console.log(sum(10, 20)); // 30

Parameter Destructuring

You can use object and array destructuring in function parameters. 

// Object destructuring in parameters
function printUserInfo({ name, age, email = 'N/A' }) {
  console.log(`Name: ${name}, Age: ${age}, Email: ${email}`);
}

printUserInfo({ name: 'John', age: 30 });
// "Name: John, Age: 30, Email: N/A"

// Array destructuring in parameters
function getFirstAndLast([first, ...rest]) {
  const last = rest.pop() || first;
  return [first, last];
}

console.log(getFirstAndLast([1, 2, 3, 4, 5])); // [1, 5]

The arguments Object

Regular functions (not arrow functions) have access to an arguments object containing all passed arguments. 

function oldSchoolSum() {
  let total = 0;
  for (let i = 0; i < arguments.length; i++) {
    total += arguments[i];
  }
  return total;
}

console.log(oldSchoolSum(1, 2, 3, 4)); // 10

// Note: Rest parameters are preferred in modern code

Function Scope

Functions in JavaScript create their own scope, which determines the accessibility of variables defined within the function.

javascript
// Global scope
const globalVar = 'I am global';

function outerFunction() {
  // Function scope
  const outerVar = 'I am from outer function';
  
  console.log(globalVar); // "I am global" - accessible
  
  function innerFunction() {
    // Nested function scope
    const innerVar = 'I am from inner function';
    
    console.log(globalVar); // "I am global" - accessible
    console.log(outerVar); // "I am from outer function" - accessible
    console.log(innerVar); // "I am from inner function" - accessible
  }
  
  innerFunction();
  
  // innerVar is not accessible here
  // console.log(innerVar); // ReferenceError
}

outerFunction();

// outerVar and innerVar are not accessible here
// console.log(outerVar); // ReferenceError
// console.log(innerVar); // ReferenceError

Lexical Scope

JavaScript uses lexical scoping, which means that functions are executed using the variable scope that was in effect when they were defined, not when they are invoked. 

function createCounter() {
  let count = 0; // This variable is "captured" by the inner function
  
  return function() {
    count++;
    return count;
  };
}

const counter = createCounter();
console.log(counter()); // 1
console.log(counter()); // 2
console.log(counter()); // 3

// The count variable is not directly accessible
// console.log(count); // ReferenceError

Block Scope vs. Function Scope

Variables declared with var are function-scoped, while variables declared with let and const are block-scoped. 

function scopeExample() {
  if (true) {
    var functionScoped = 'I am function-scoped';
    let blockScoped = 'I am block-scoped';
  }
  
  console.log(functionScoped); // "I am function-scoped" - accessible
  // console.log(blockScoped); // ReferenceError - not accessible
}

scopeExample();

Closures

A closure is the combination of a function and the lexical environment within which that function was declared. This allows a function to access variables from its outer scope even after that scope has closed.

javascript
// Basic closure example
function createGreeting(greeting) {
  // The inner function has access to the greeting parameter
  return function(name) {
    return `${greeting}, ${name}!`;
  };
}

const sayHello = createGreeting('Hello');
const sayHi = createGreeting('Hi');

console.log(sayHello('John')); // "Hello, John!"
console.log(sayHi('Jane')); // "Hi, Jane!"

// Even though createGreeting has finished executing,
// the inner functions still have access to the greeting parameter

Practical Uses of Closures

Closures are powerful and have many practical applications in JavaScript:

  • Data Privacy: Creating private variables and methods
  • Function Factories: Creating specialized functions
  • Memoization: Caching function results for better performance
  • Callbacks: Preserving state in asynchronous operations
javascript
// Data privacy example
function createCounter() {
  let count = 0; // Private variable
  
  return {
    increment() {
      count++;
      return count;
    },
    decrement() {
      count--;
      return count;
    },
    getCount() {
      return count;
    }
  };
}

const counter = createCounter();
console.log(counter.increment()); // 1
console.log(counter.increment()); // 2
console.log(counter.getCount()); // 2
console.log(counter.decrement()); // 1

// The count variable is not directly accessible
// console.log(count); // ReferenceError

Higher-Order Functions

Higher-order functions are functions that operate on other functions, either by taking them as arguments or by returning them. They are a powerful feature of JavaScript that enables functional programming patterns.

Functions as Arguments

Functions can be passed as arguments to other functions. 

function executeOperation(x, y, operation) {
  return operation(x, y);
}

function add(a, b) {
  return a + b;
}

function multiply(a, b) {
  return a * b;
}

console.log(executeOperation(5, 3, add)); // 8
console.log(executeOperation(5, 3, multiply)); // 15

Functions as Return Values

Functions can return other functions. 

function multiplier(factor) {
  return function(number) {
    return number * factor;
  };
}

const double = multiplier(2);
const triple = multiplier(3);

console.log(double(5)); // 10
console.log(triple(5)); // 15

Array Methods as Higher-Order Functions

Many built-in array methods in JavaScript are higher-order functions that accept callback functions. 

const numbers = [1, 2, 3, 4, 5];

// map: transforms each element
const squared = numbers.map(num => num * num);
console.log(squared); // [1, 4, 9, 16, 25]

// filter: selects elements that match a condition
const evenNumbers = numbers.filter(num => num % 2 === 0);
console.log(evenNumbers); // [2, 4]

// reduce: accumulates values
const sum = numbers.reduce((total, num) => total + num, 0);
console.log(sum); // 15

// forEach: executes a function for each element
numbers.forEach(num => console.log(num * 2));
// Outputs: 2, 4, 6, 8, 10

Best Practices

Following these best practices will help you write cleaner, more maintainable JavaScript functions.

Do

  • Keep functions small and focused

    Each function should do one thing and do it well

  • Use descriptive function names

    Names should clearly indicate what the function does

  • Use default parameters when appropriate

    Makes functions more flexible and easier to use

  • Return early to avoid deep nesting

    Improves readability and reduces complexity

  • Use arrow functions for short callbacks

    Makes code more concise and readable

Don't

  • Avoid side effects when possible

    Functions should not modify variables outside their scope unexpectedly

  • Don't use too many parameters

    Consider using an object parameter instead

  • Avoid using the arguments object

    Use rest parameters instead for better readability

  • Don't declare functions inside loops

    Can lead to performance issues and unexpected behavior

  • Avoid using eval()

    Poses security risks and performance issues

javascript
// Good practices

// 1. Small, focused function
function calculateTax(amount, taxRate = 0.1) {
  return amount * taxRate;
}

// 2. Descriptive name and early return
function isValidUsername(username) {
  if (!username) return false;
  if (username.length < 3) return false;
  if (username.length > 20) return false;
  
  return /^[a-zA-Z0-9_]+$/.test(username);
}

// 3. Object parameter for many options
function createUser({ username, email, age, isAdmin = false }) {
  // Create user logic
  return { username, email, age, isAdmin };
}

// 4. Arrow function for short callbacks
const numbers = [1, 2, 3, 4, 5];
const doubled = numbers.map(num => num * 2);

Pure Functions

A pure function is a function that:

  • Always returns the same output for the same input
  • Has no side effects (doesn't modify external state)

Pure functions are easier to test, debug, and reason about. They're a key concept in functional programming. 

// Pure function
function add(a, b) {
  return a + b;
}

// Impure function (has side effects)
let total = 0;
function addToTotal(value) {
  total += value; // Side effect: modifies external state
  return total;
}

Common Function Patterns

These common function patterns and techniques will help you write more effective JavaScript code.

Callback Pattern

A callback is a function passed as an argument to another function, which is then invoked inside the outer function.

javascript
// Callback pattern example
function fetchData(url, callback) {
  console.log(`Fetching data from ${url}...`);
  
  // Simulate async operation
  setTimeout(() => {
    const data = { id: 1, name: 'John Doe' };
    callback(data);
  }, 1000);
}

fetchData('https://api.example.com/users', (data) => {
  console.log('Data received:', data);
});

Immediately Invoked Function Expression (IIFE)

An IIFE is a function that runs as soon as it is defined. It's often used to create a private scope.

javascript
// IIFE pattern
(function() {
  // Private scope
  const privateVar = 'I am private';
  console.log(privateVar);
})();

// With parameters
(function(name) {
  console.log(`Hello, ${name}!`);
})('John');

// Arrow function IIFE
((x, y) => {
  console.log(x + y);
})(10, 20);

Currying

Currying is the technique of transforming a function that takes multiple arguments into a sequence of functions that each take a single argument.

javascript
// Currying example
function multiply(a) {
  return function(b) {
    return a * b;
  };
}

// Using the curried function
const multiplyByTwo = multiply(2);
console.log(multiplyByTwo(4)); // 8
console.log(multiplyByTwo(10)); // 20

// Arrow function version
const add = a => b => a + b;
const addFive = add(5);
console.log(addFive(3)); // 8

Memoization

Memoization is a technique used to speed up function calls by caching the results of expensive function calls.

javascript
// Memoization example
function memoize(fn) {
  const cache = {};
  
  return function(...args) {
    const key = JSON.stringify(args);
    
    if (cache[key]) {
      console.log('Returning from cache');
      return cache[key];
    }
    
    const result = fn.apply(this, args);
    cache[key] = result;
    return result;
  };
}

// Expensive function
function fibonacci(n) {
  if (n <= 1) return n;
  return fibonacci(n - 1) + fibonacci(n - 2);
}

// Memoized version
const memoizedFibonacci = memoize(function(n) {
  if (n <= 1) return n;
  return memoizedFibonacci(n - 1) + memoizedFibonacci(n - 2);
});

console.time('Regular');
fibonacci(30);
console.timeEnd('Regular');

console.time('Memoized');
memoizedFibonacci(30);
console.timeEnd('Memoized');
