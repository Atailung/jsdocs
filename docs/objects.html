Object Basics

Objects are collections of related data and functionality, stored as key-value pairs. They are one of the most fundamental and powerful data structures in JavaScript.

Why Objects Matter

Objects allow us to:

  • Store related data together in a structured way
  • Model real-world entities with properties and behaviors
  • Create reusable templates for data through prototypes
  • Organize code in a modular, maintainable way
user
name: "John Doe"
age: 30
isActive: true
greet: function() {...}

In JavaScript, objects consist of properties, which are key-value pairs. The keys (or property names) are strings or symbols, and the values can be any data type, including other objects, arrays, or functions (which are then called methods).

javascript
// A simple object with various property types
const user = {
  name: 'John Doe',
  age: 30,
  isActive: true,
  address: {
    street: '123 Main St',
    city: 'New York',
    zipCode: '10001'
  },
  hobbies: ['reading', 'coding', 'hiking'],
  greet: function() {
    return `Hello, my name is ${this.name}`;
  }
};

// Accessing object properties
console.log(user.name); // 'John Doe'
console.log(user.address.city); // 'New York'
console.log(user.hobbies[1]); // 'coding'

// Calling an object method
console.log(user.greet()); // 'Hello, my name is John Doe'

Creating Objects

There are several ways to create objects in JavaScript. Let's explore the different methods.

Object Literal

The most common way to create an object is using curly braces {}. 

// Empty object
const emptyObject = {};

// Object with properties
const person = {
  name: 'Alice',
  age: 28,
  isEmployed: true
};

Object Constructor

You can use the Object() constructor to create objects. 

// Empty object
const emptyObject = new Object();

// Adding properties
emptyObject.name = 'Bob';
emptyObject.age = 32;
emptyObject.isEmployed = false;= false;

Constructor Function

Constructor functions create objects with a specific structure and behavior. 

// Constructor function
function Person(name, age, isEmployed) {
  this.name = name;
  this.age = age;
  this.isEmployed = isEmployed;
  this.greet = function() {
    return `Hello, I'm ${this.name}`;
  };
}

// Creating instances
const person1 = new Person('Charlie', 35, true);
const person2 = new Person('Diana', 27, true);

ES6 Classes

Modern JavaScript uses classes to create objects with a cleaner syntax. 

// Class definition
class Person {
  constructor(name, age, isEmployed) {
    this.name = name;
    this.age = age;
    this.isEmployed = isEmployed;
  }
  
  greet() {
    return `Hello, I'm ${this.name}`;
  }
}

// Creating instances
const person1 = new Person('Eve', 29, false);

Object.create()

Creates a new object with the specified prototype object and properties. 

// Create a prototype object
const personProto = {
  greet() {
    return `Hello, I'm ${this.name}`;
  }
};

// Create a new object with the prototype
const person = Object.create(personProto);
person.name = 'Frank';
person.age = 31;

console.log(person.greet()); // "Hello, I'm Frank"

Factory Function

Functions that create and return new objects without using new. 

// Factory function
function createPerson(name, age, isEmployed) {
  return {
    name,  // Shorthand for name: name
    age,
    isEmployed,
    greet() {
      return `Hello, I'm ${this.name}`;
    }
  };
}

const person1 = createPerson('Grace', 33, true);
const person2 = createPerson('Henry', 26, false);

Choosing the Right Method

Each method of creating objects has its own use cases:

  • Object Literals

    Best for one-off objects or when you need a simple data structure.

  • Constructor Functions / Classes

    Ideal when you need multiple instances with the same structure and behavior.

  • Object.create()

    Useful when you want fine-grained control over object inheritance.

  • Factory Functions

    Great for creating objects with private data through closures.

Properties & Methods

Objects in JavaScript consist of properties and methods. Properties are key-value pairs, and methods are functions stored as property values.

Properties

Properties are the data stored in an object. They can be of any data type. 

const person = {
  // String property
  name: 'John Doe',
  
  // Number property
  age: 30,
  
  // Boolean property
  isEmployed: true,
  
  // Array property
  hobbies: ['reading', 'coding'],
  
  // Object property (nested object)
  address: {
    street: '123 Main St',
    city: 'New York'
  }
};

Methods

Methods are functions stored as object properties. They can access and modify the object's data. 

const calculator = {
  // Data properties
  value: 0,
  
  // Method using function declaration
  add: function(num) {
    this.value += num;
    return this.value;
  },
  
  // Method using shorthand syntax (ES6)
  subtract(num) {
    this.value -= num;
    return this.value;
  },
  
  // Method using arrow function
  reset: () => {
    this.value = 0;
    return this.value;
  }
};

The 'this' Keyword

In object methods, this refers to the object the method belongs to. It allows methods to access and modify the object's properties. 

const person = {
  firstName: 'John',
  lastName: 'Doe',
  
  // Method using this to access other properties
  getFullName() {
    return `${this.firstName} ${this.lastName}`;
  },
  
  setName(first, last) {
    this.firstName = first;
    this.lastName = last;
  }
};

console.log(person.getFullName()); // "John Doe"
person.setName('Jane', 'Smith');
console.log(person.getFullName()); // "Jane Smith"

Note: Arrow functions don't have their own this context. They inherit this from the surrounding scope, which can cause unexpected behavior in object methods.

Accessing Object Data

There are multiple ways to access properties and methods in JavaScript objects.

javascript
const person = {
  name: 'John Doe',
  age: 30,
  address: {
    street: '123 Main St',
    city: 'New York'
  },
  hobbies: ['reading', 'coding', 'hiking'],
  greet() {
    return `Hello, my name is ${this.name}`;
  }
};

// 1. Dot Notation
console.log(person.name); // 'John Doe'
console.log(person.address.city); // 'New York'
console.log(person.hobbies[1]); // 'coding'
console.log(person.greet()); // 'Hello, my name is John Doe'

// 2. Bracket Notation
console.log(person['name']); // 'John Doe'
console.log(person['address']['city']); // 'New York'
console.log(person['hobbies'][1]); // 'coding'

// Using variables with bracket notation
const propertyName = 'age';
console.log(person[propertyName]); // 30

Dot Notation

The most common way to access object properties.

Advantages:

  • Cleaner, more readable syntax
  • Better autocomplete in IDEs
  • Slightly faster performance

Limitations:

  • Cannot use variables as property names
  • Cannot use property names with spaces or special characters
  • Cannot use property names that start with a number

Bracket Notation

More flexible but slightly more verbose.

Advantages:

  • Can use variables as property names
  • Can access properties with spaces or special characters
  • Can access properties that start with a number
  • Can use expressions to compute property names

Example with dynamic access:

const user = {
  name: 'Alice',
  age: 28,
  'user-id': 'ABC123'
};

function getProperty(obj, prop) {
  return obj[prop];
}

console.log(getProperty(user, 'name')); // 'Alice'
console.log(user['user-id']); // 'ABC123'

Optional Chaining

The optional chaining operator (?.) allows you to access deeply nested object properties without worrying about whether intermediate properties exist. 

const user = {
  name: 'John',
  address: {
    city: 'New York'
  }
};

// Without optional chaining - can cause errors
// console.log(user.employment.company); // Error: Cannot read property 'company' of undefined

// With optional chaining - safe access
console.log(user?.employment?.company); // undefined (no error)
console.log(user?.address?.city); // 'New York'

// Works with methods too
console.log(user?.getDetails?.()); // undefined (no error)

Modifying Objects

Objects in JavaScript are mutable, meaning you can change their properties after creation.

javascript
const person = {
  name: 'John Doe',
  age: 30
};

// Adding new properties
person.email = 'john@example.com';
person['phone'] = '555-1234';

// Modifying existing properties
person.age = 31;
person['name'] = 'John Smith';

// Adding nested objects
person.address = {
  street: '123 Main St',
  city: 'New York'
};

// Deleting properties
delete person.phone;

console.log(person);

Adding & Modifying Properties

You can add new properties or modify existing ones using dot or bracket notation. 

const user = {
  name: 'Alice'
};

// Adding properties
user.age = 28;
user['isActive'] = true;

// Adding a method
user.greet = function() {
  return `Hello, I'm ${this.name}`;
};

// Modifying properties
user.name = 'Alicia';

Deleting Properties

The delete operator removes a property from an object. 

const user = {
  name: 'Bob',
  age: 32,
  email: 'bob@example.com',
  phone: '555-5678'
};

// Delete a property
delete user.phone;

// Delete using bracket notation
delete user['email'];

console.log(user);
// { name: 'Bob', age: 32 }

Object Immutability

While objects declared with const can have their properties modified, sometimes you want to prevent modifications to ensure data integrity. 

// Object.freeze() - makes an object completely immutable
const frozenUser = Object.freeze({
  name: 'Charlie',
  age: 35
});

// These will fail silently in non-strict mode
// or throw errors in strict mode
frozenUser.age = 36;
frozenUser.email = 'charlie@example.com';
delete frozenUser.name;

console.log(frozenUser);
// Still { name: 'Charlie', age: 35 }

// Object.seal() - prevents adding/deleting properties
// but allows modifying existing properties
const sealedUser = Object.seal({
  name: 'Diana',
  age: 27
});

// This works
sealedUser.age = 28;

// These will fail
sealedUser.email = 'diana@example.com';
delete sealedUser.name;

Object Methods

JavaScript provides several built-in methods for working with objects through the Object constructor.

Object.keys(), values(), entries()

Methods for getting arrays of an object's keys, values, or key-value pairs. 

const person = {
  name: 'John',
  age: 30,
  city: 'New York'
};

// Get all keys
const keys = Object.keys(person);
console.log(keys);
// ['name', 'age', 'city']

// Get all values
const values = Object.values(person);
console.log(values);
// ['John', 30, 'New York']

// Get all entries (key-value pairs)
const entries = Object.entries(person);
console.log(entries);
// [['name', 'John'], ['age', 30], ['city', 'New York']]

Object.assign()

Copies properties from one or more source objects to a target object. 

const target = { a: 1, b: 2 };
const source1 = { b: 3, c: 4 };
const source2 = { c: 5, d: 6 };

// Copy properties from sources to target
const result = Object.assign(target, source1, source2);

console.log(target);
// { a: 1, b: 3, c: 5, d: 6 }

console.log(result === target);
// true (result is the modified target)

// Creating a new object without modifying sources
const newObj = Object.assign({}, target, source1);

hasOwnProperty()

Checks if an object has a specific property as its own (not inherited). 

const person = {
  name: 'John',
  age: 30
};

console.log(person.hasOwnProperty('name'));
// true

console.log(person.hasOwnProperty('toString'));
// false (toString is inherited from Object.prototype)

// Safer alternative in modern code
console.log(Object.hasOwn(person, 'name'));
// true

Object.is()

Determines if two values are the same value. 

// Similar to === but with some differences
console.log(Object.is(5, 5));
// true

console.log(Object.is(5, '5'));
// false

// Differences from ===
console.log(Object.is(0, -0));
// false (=== would return true)

console.log(Object.is(NaN, NaN));
// true (=== would return false)

Spread Operator with Objects

The spread operator (...) provides a concise way to copy properties from one object to another. 

const person = {
  name: 'John',
  age: 30
};

// Creating a copy of an object
const personCopy = { ...person };
console.log(personCopy);
// { name: 'John', age: 30 }

// Merging objects (similar to Object.assign)
const details = { job: 'Developer', city: 'New York' };
const completeProfile = { ...person, ...details };
console.log(completeProfile);
// { name: 'John', age: 30, job: 'Developer', city: 'New York' }

// Overriding properties
const updatedPerson = { ...person, age: 31, email: 'john@example.com' };
console.log(updatedPerson);
// { name: 'John', age: 31, email: 'john@example.com' }

Object Destructuring

Object destructuring is a convenient way to extract values from objects and assign them to variables.

javascript
// Basic destructuring
const person = {
  name: 'John Doe',
  age: 30,
  job: 'Developer',
  address: {
    city: 'New York',
    country: 'USA'
  }
};

const { name, age, job } = person;

console.log(name);  // 'John Doe'
console.log(age);   // 30
console.log(job);   // 'Developer'

Renaming Variables

You can assign object properties to variables with different names. 

const person = {
  name: 'John Doe',
  age: 30
};

// Rename variables during destructuring
const { name: fullName, age: years } = person;

console.log(fullName); // 'John Doe'
console.log(years);    // 30

// name and age are not defined
// console.log(name); // ReferenceError

Default Values

You can provide fallback values for properties that might not exist. 

const person = {
  name: 'John Doe',
  age: 30
};

// Provide default values
const { 
  name, 
  age, 
  job = 'Unknown',
  email = 'no-email@example.com'
} = person;

console.log(job);   // 'Unknown'
console.log(email); // 'no-email@example.com'

Nested Destructuring

You can destructure nested objects to extract deeper properties. 

const person = {
  name: 'John Doe',
  age: 30,
  address: {
    street: '123 Main St',
    city: 'New York',
    country: 'USA'
  }
};

// Destructuring nested objects
const { 
  name, 
  address: { city, country } 
} = person;

console.log(name);    // 'John Doe'
console.log(city);    // 'New York'
console.log(country); // 'USA'

// address is not defined
// console.log(address); // ReferenceError

Function Parameters

Destructuring is particularly useful in function parameters. 

// Without destructuring
function printUserInfo(user) {
  console.log(`Name: ${user.name}, Age: ${user.age}`);
}

// With destructuring
function printUserInfo({ name, age, job = 'Unknown' }) {
  console  'Unknown' }) {
  console.log(`Name: ${name}, Age: ${age}, Job: ${job}`);
}

// Using the function
printUserInfo({ name: 'Alice', age: 28 });
// "Name: Alice, Age: 28, Job: Unknown"

// With default parameter
function createUser({ name, age } = { name: 'Guest', age: 0 }) {

Prototypes & Inheritance

JavaScript uses a prototype-based inheritance model, where objects can inherit properties and methods from other objects.

Understanding Prototypes

Every JavaScript object has a prototype, which is another object that it inherits properties and methods from. This creates a prototype chain that ends with Object.prototype. 

// Creating an object
const person = {
  name: 'John',
  greet() {
    return `Hello, I'm ${this.name}`;
  }
};

// person's prototype is Object.prototype
console.log(Object.getPrototypeOf(person) === Object.prototype);
// true

// Accessing a method from the prototype chain
console.log(person.toString()); // "[object Object]"
// toString is inherited from Object.prototype

Prototype Inheritance

Creating inheritance relationships between objects. 

// Parent object
const animal = {
  type: 'Animal',
  makeSound() {
    return 'Some generic sound';
  }
};

// Child object inheriting from animal
const dog = Object.create(animal);
dog.breed = 'Labrador';
dog.makeSound = function() {
  return 'Woof!';
};

console.log(dog.type); // 'Animal' (inherited)
console.log(dog.breed); // 'Labrador'
console.log(dog.makeSound()); // 'Woof!' (overridden)

// Check the prototype chain
console.log(Object.getPrototypeOf(dog) === animal);
// true

Constructor Functions & Prototypes

Using constructor functions to create objects with shared methods. 

// Constructor function
function Person(name, age) {
  this.name = name;
  this.age = age;
}

// Adding a method to the prototype
Person.prototype.greet = function() {
  return `Hello, I'm ${this.name}`;
};

// Creating instances
const person1 = new Person('John', 30);
const person2 = new Person('Jane', 25);

console.log(person1.greet()); // "Hello, I'm John"
console.log(person2.greet()); // "Hello, I'm Jane"

// Both instances share the same method
console.log(person1.greet === person2.greet);
// true

ES6 Classes

ES6 introduced a class syntax that provides a cleaner way to work with prototypes and inheritance. 

// Parent class
class Animal {
  constructor(name) {
    this.name = name;
  }
  
  makeSound() {
    return 'Some generic sound';
  }
}

// Child class extending Animal
class Dog extends Animal {
  constructor(name, breed) {
    super(name); // Call parent constructor
    this.breed = breed;
  }
  
  makeSound() {
    return 'Woof!';
  }
  
  fetch() {
    return `${this.name} is fetching`;
  }
}

const dog = new Dog('Rex', 'German Shepherd');

console.log(dog.name);        // 'Rex'
console.log(dog.breed);       // 'German Shepherd'
console.log(dog.makeSound()); // 'Woof!'
console.log(dog.fetch());     // 'Rex is fetching'

// Checking inheritance
console.log(dog instanceof Dog);    // true
console.log(dog instanceof Animal); // true

Best Practices

Following these best practices will help you write cleaner, more maintainable JavaScript code when working with objects.

Do

  • Use object literals for simple data structures

    They're concise and easy to read

  • Use classes for complex objects with methods

    Provides a cleaner syntax for inheritance and shared methods

  • Use property shorthand when variable names match property names

    { name, age } instead of { name: name, age: age }

  • Use computed property names for dynamic keys

    { [dynamicKey]: value }

  • Use destructuring to extract properties

    Makes code cleaner and more readable

Don't

  • Avoid modifying built-in object prototypes

    Can lead to conflicts with other code and future JavaScript features

  • Don't use arrow functions for object methods that use this

    Arrow functions don't have their own this context

  • Avoid using delete on object properties

    It's slow and can change the object's hidden class in engines

  • Don't use for...in without hasOwnProperty checks

    It iterates over inherited properties too

  • Avoid creating deeply nested object structures

    Makes code harder to read and maintain

javascript
// Good practices

// 1. Use object literals for simple data
const user = {
  name: 'John',
  age: 30
};

// 2. Use property shorthand
const name = 'Alice';
const age = 25;
const person = { name, age }; // Instead of { name: name, age: age }

// 3. Use computed property names
const propertyName = 'email';
const contact = {
  [propertyName]: 'john@example.com'
};

// 4. Use classes for complex objects
class Product {
  constructor(name, price) {
    this.name = name;
    this.price = price;
  }
  
  getDiscountedPrice(discount) {
    return this.price * (1 - discount);
  }
}

// 5. Use Object.assign or spread for shallow copies
const original = { a: 1, b: 2 };
const copy1 = Object.assign({}, original);
const copy2 = { ...original };

Performance Considerations

When working with objects, keep these performance tips in mind:

  • Initialize objects with all properties they'll need to avoid hidden class changes
  • Use Object.keys(), Object.values(), or Object.entries() instead of for...in loops
  • Set properties to null instead of using delete when you want to remove them
  • Be careful with deep object cloning, as it can be expensive for large objects
  • Use object pooling for frequently created and discarded objects in performance-critical code

Common Object Patterns

These common object patterns and techniques will help you write more effective JavaScript code.

Object Composition

Combining simple objects to create more complex ones.

javascript
// Creating behavior objects
const canEat = {
  eat(food) {
    console.log(`Eating ${food}`);
  }
};

const canSleep = {
  sleep(hours) {
    console.log(`Sleeping for ${hours} hours`);
  }
};

const canCode = {
  code(language) {
    console.log(`Coding in ${language}`);
  }
};

// Creating a person with composed behaviors
function createPerson(name) {
  return {
    name,
    ...canEat,
    ...canSleep
  };
}

// Creating a programmer with additional behaviors
function createProgrammer(name) {
  return {
    ...createPerson(name),
    ...canCode
  };
}

const person = createPerson('John');
person.eat('pizza');
person.sleep(8);

const programmer = createProgrammer('Jane');
programmer.eat('sushi');
programmer.code('JavaScript');

Module Pattern

Creating private variables and methods using closures.

javascript
const counter = (function() {
  // Private variables
  let count = 0;
  
  // Private function
  function validate(n) {
    return typeof n === 'number' && Number.isInteger(n);
  }
  
  // Public interface
  return {
    increment() {
      count++;
      return count;
    },
    decrement() {
      count--;
      return count;
    },
    getValue() {
      return count;
    },
    setCount(n) {
      if (validate(n)) {
        count = n;
        return true;
      }
      return false;
    }
  };
})();

console.log(counter.getValue()); // 0
console.log(counter.increment()); // 1
console.log(counter.increment()); // 2
console.log(counter.decrement()); // 1
console.log(counter.setCount(10)); // true
console.log(counter.getValue()); // 10

// Private variables and methods are not accessible
// console.log(counter.count); // undefined
// console.log(counter.validate); // undefined

Factory Pattern

Creating objects with a factory function.

javascript
// User factory function
function createUser({ name, age, role = 'user' }) {
  return {
    name,
    age,
    role,
    createdAt: new Date(),
    describe() {
      return `${this.name} is ${this.age} years old and has the role of ${this.role}`;
    },
    hasPermission(permission) {
      const permissions = {
        user: ['read'],
        editor: ['read', 'write'],
        admin: ['read', 'write', 'delete']
      };
      return permissions[this.role].includes(permission);
    }
  };
}

const user1 = createUser({ name: 'John', age: 30 });
const user2 = createUser({ name: 'Jane', age: 28, role: 'admin' });

console.log(user1.describe());
console.log(user1.hasPermission('read')); // true
console.log(user1.hasPermission('write')); // false

console.log(user2.describe());
console.log(user2.hasPermission('delete')); // true

Singleton Pattern

Ensuring only one instance of an object exists.

javascript
const Database = (function() {
  let instance;
  
  // Private constructor function
  function createInstance() {
    const object = {
      data: {},
      insert(key, value) {
        this.data[key] = value;
      },
      get(key) {
        return this.data[key];
      },
      delete(key) {
        delete this.data[key];
      }
    };
    return object;
  }
  
  return {
    // Public method to get the instance
    getInstance() {
      if (!instance) {
        instance = createInstance();
      }
      return instance;
    }
  };
})();

// Usage
const db1 = Database.getInstance();
const db2 = Database.getInstance();

console.log(db1 === db2); // true - same instance

db1.insert('user', { name: 'John' });
console.log(db2.get('user')); // { name: 'John' }
